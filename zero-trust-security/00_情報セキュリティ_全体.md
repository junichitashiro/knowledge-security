# 情報セキュリティ 全体俯瞰

## 1. 情報セキュリティの基礎

### 構成要素（基本3要素）
- 機密性 / Confidentiality
- 完全性 / Integrity
- 可用性 / Availability

### 構成要素（追加4要素）
- 真正性 / Authenticity
- 責任追跡性 / Accountability
- 否認防止 / Non-Repudiation
- 信頼性 / Reliability

### 情報資産
- 紙・電子情報
- ハードウェア
- サービス
- 人
- 無形資産

### 脅威
- 物理的脅威
- 技術的脅威
- 人的脅威

### 脆弱性
- 物理的脆弱性
- 技術的脆弱性
- 人的脆弱性

## 2. リスク対策の種類

### リスクコントロール
- リスク回避（確率: 高 / 被害: 高）
- リスク最適化（確率: 高 / 被害: 低）

### リスクファイナンス
- リスク転移（確率: 低 / 被害: 高）
- リスク保有（確率: 低 / 被害: 低）

## 3. マルウェア

### 語源
- Malicious + Software

### 分類
- トロイの木馬
- ウィルス
- ワーム
- ランサムウェア
- スケアウェア

## 4. サイバー攻撃のプロセス

### Cyber Kill Chain
- 1. 偵察
- 2. 武器化
- 3. 配送
- 4. 攻撃
- 5. インストール
- 6. 遠隔操作
- 7. 目的実行

### ATT&CK for Enterprise
- Cyber Kill Chain より詳細
- 戦術・テクニックをカバー

## 5. 境界型セキュリティ

### 考え方
- 外部と内部の境界で侵入チェック
- 内部は信頼する

### 多層防御
- 入り口対策（FW / IPS / IDS / ウィルス対策）
- 内部対策（暗号化 / ログ監視）
- 出口対策（漏洩対策 / FW / ネットワーク監視）

### 最小権限の原則

### 課題
- 攻撃の巧妙化
- 被害拡大
- VPNの枯渇
- 出口の負荷集中
- クラウド移行
- グローバル化

## 6. ゼロトラストセキュリティ

### きっかけ
- Google BeyondCorp 論文

### 考え方
- すべてを疑う
- 境界型をベースに組み合わせる

### 認証と認可
- 認証: MFA / RBA
- 認可: 必要最低限・アプリ単位・データ単位

### 信頼と信用
- 信頼 / trust（主観的）
- 信用 / credit（客観的）

### NIST 7原則
- 1. リソースの定義
- 2. 通信の保護
- 3. セッション単位のアクセス
- 4. 動的ポリシー
- 5. デバイス監視
- 6. 動的な認証・認可
- 7. 情報収集と活用

## 7. 構成要素

### IAM
- 認証（MFA / RBA）
- 認可（SSO）
- IDデータベース
- ポリシー

### IAP
- アプリ単位のアクセス制御
- VPNより柔軟・安全

### PKI
- 共通鍵暗号方式
- 公開鍵暗号方式
- mTLS

### SWG
- Webフィルタリング
- アンチウィルス
- DLP

### TPM
- 端末レベルの暗号化・認証
- BitLocker で利用

### MDM・MAM
- MDM: 端末を遠隔管理
- MAM: 業務領域と個人領域を区分

### DLP
- 重要情報の監視・漏洩防止
- 経路: メール / チャット / SaaS / USB / 印刷

### SIEM・SOAR
- SIEM: ログ収集と脅威検知
- SOAR: インシデント管理・自動対応
