## IAM（Identity and Access Management）― IDアクセス管理

### 認証

#### 多要素認証（MFA：Multi-Factor Authentication）
複数の要素を使って認証を行うこと。二段階認証とは異なり、要素の種類が重要。

| 要素     | 種類                           | 例                      |
| -------- | ------------------------------ | ----------------------- |
| 知識情報 | 本人が知っている情報           | パスワード、秘密の質問  |
| 所持情報 | 本人が持っているものにある情報 | ワンタイムトークン、SMS |
| 生体情報 | 本人の生体に関わる情報         | 指紋、静脈、虹彩        |

#### リスクベース認証（RBA：Risk Based Authentication）
- 多要素認証に追加して行う認証。IAMの一部。
- 金融系で昔から使われてきた。
- 普段の行動パターンを分析し、リスクに応じて追加認証を要求する（学習期間が必要）。
- **リスク例**：普段と異なるログイン場所、匿名IPアドレスからのログイン

### 認可
- SSO（シングルサインオン）による一元管理

### IDデータベース
- ユーザーID・氏名・所属を管理するDB
- 社内では人事システム等と連携

### ポリシー
- 利用できるアプリケーションのルールを定義する

## IAP（Identity Aware Proxy）― ID認識型プロキシ

端末を保護しつつ、クラウドとオンプレミスへのアクセスを実現する。IAPがIAMへ問い合わせて認証を行う。コネクターを導入することで穴あけせずにオンプレミスに接続可能。

### VPNとの比較

| 観点             | VPN                                                  | IAP                                            |
| ---------------- | ---------------------------------------------------- | ---------------------------------------------- |
| 管理単位         | ネットワーク全体                                     | アプリケーション単位（被害もアプリ単位に限定） |
| ネットワーク構成 | アクセスポイントの穴あけが必要（設定不備リスクあり） | コネクターの導入だけ（穴あけ不要）             |
| 柔軟性           | ハードウェア・回線業者から調達が必要                 | クラウドサービスのためアクセス負荷の考慮が不要 |

### セキュリティ思想の比較

| 思想               | 考え方                               |
| ------------------ | ------------------------------------ |
| 境界型セキュリティ | 外側は危険、内側は安全               |
| ゼロトラスト       | すべてを疑う、すべての通信を保護する |

## PKI（Public Key Infrastructure）― 公開鍵基盤

### 暗号化方式

#### 共通鍵暗号方式
- 暗号化・復号の両方に同じ共通鍵を使用する

#### 公開鍵暗号方式
- 暗号化：秘密鍵（本人の真正性を示す）
- 復号：公開鍵

### TLS相互認証（mTLS：Mutual-TLS）
お互いを認証する仕組み。

- **クライアント**：クライアント証明書を持つ
- **サーバー**：サーバー証明書を持つ

**課題**：
- クライアント側は利用するサーバーごとに証明書が必要
- 公的な認証局を利用すると費用が膨大になる

**対策**：プライベート認証局を利用する

## SWG（Secure Web Gateway）― セキュアWebゲートウェイ

### 役割

| 機能                | 内容                           |
| ------------------- | ------------------------------ |
| Webフィルタリング   | 危険なサイトへのアクセスを遮断 |
| アンチウィルス      | マルウェア検査・侵入防止       |
| 情報漏洩防止（DLP） | 許可しないアップロードを禁止   |

### 特徴
- 社内プロキシの負荷分散
- 必ずしも外へ出るためのプロキシを経由しない

## TPM（Trusted Platform Module）― セキュリティチップ

コンピュータのセキュリティ・プライバシーを強化する暗号化モジュール。クライアント証明書を安全に保管する仕組み。Windows 11では必須。

### できること
- 暗号化・復号
- 公開鍵暗号のキーペア作成
- 鍵情報の保管・使用制限
- プラットフォームの改ざん検知

### 特徴
- OSと独立しているため、OSの脆弱性の影響を受けない

### 利用例：BitLocker（HDD暗号化）
- TPMとHDDの組み合わせが一致しないと復号できない
- HDDだけ抜き取っても中身が見えない
- HDD暗号化に利用した鍵をTPMへ保管

## MDM・MAM ― モバイルデバイス・アプリケーション管理

### MDM（Mobile Device Management）
スマートフォンやノートPCなどを遠隔で管理する。

**仕組み**：端末にエージェントを入れて管理。IAMと連携して個人と端末を紐づける。

| 機能                   | 内容                                         |
| ---------------------- | -------------------------------------------- |
| 資産管理               | OSバージョン・アプリ・位置などのインベントリ |
| リモートロック・ワイプ | 紛失時のロック・データ削除                   |
| アプリケーション管理   | 必要なアプリの配布・利用制限                 |
| パッチ管理             | OS・アプリの更新                             |
| ポリシー管理           | パスワード・Wi-Fi・VPN・プロファイル         |

### MAM（Mobile Application Management）
モバイル端末上で業務領域と個人領域を区分して管理する。コンテナ（仮想領域）を作成して区分。

- 業務領域から個人領域へのコピーを禁止
- 業務データのみを選択的に削除できる

## DLP（Data Loss Prevention）― 情報漏洩防止

### 役割
- 重要情報の監視
- 漏洩防止

### 主な漏洩経路
メール・チャット・SaaS・USB・印刷

### 原因

| 種別         | 例                 |
| ------------ | ------------------ |
| インサイダー | 退所時の持ち出し   |
| 攻撃         | マルウェア         |
| 不注意       | メール添付の誤送信 |

### 仕組みの変遷
- **旧来**：守りたいファイルにタグをつけ、エージェントで監視する
- **最近**：クラウド型（URLフィルタだけでは防げず、ログイン情報も検査する必要がある）

## SIEM・SOAR ― セキュリティ情報管理と自動対応

### SIEM（Security Information and Event Management）
ログを収集して一元管理し、脅威を検知する。

**収集対象**：ネットワーク機器・サーバー・クラウドサービス

| 役割     | 内容                                     |
| -------- | ---------------------------------------- |
| 正規化   | 異なる形式のログを統一フォーマットに変換 |
| 相関分析 | 複数ログをまたいで脅威パターンを分析     |
| 可視化   | ダッシュボードで状況を把握               |
| 脅威検知 | 異常を検知してアラートを発報             |

- **UEBA**（User and Entity Behavior Analytics）：人手では難しいため機械学習を活用する

### SOAR（Security Orchestration, Automation and Response）
- SIEMで見つかったインシデントを管理する
- 対処を自動化することで対応速度を向上させる
