## きっかけ

Googleによる **BeyondCorpの研究論文** が発端。境界型セキュリティの限界を示唆した。

---

## 考え方

- **すべてを疑う**
- 境界型セキュリティを否定するものではなく、むしろベースになる
- 「導入すれば安全が実現する」サービスや製品は存在しない
- システムの構成に合わせて防御方法を組み合わせて構成する

---

## 認証と認可

### 認証（人を特定すること）
- 多要素認証（MFA）
- リスクベース認証（RBA）

### 認可（アクセス権を特定すること）
- 必要最低限が基本
- アプリケーション単位での制御
- データ単位での制御

## 信頼と信用

|      | 信頼（trust）                | 信用（credit）   |
| ---- | ---------------------------- | ---------------- |
| 性質 | 主観的                       | 客観的           |
| 定義 | 信じて頼ること・信じる気持ち | 行為・業績・評価 |
| 関係 | 信用の積み重ねから信頼を得る | 信頼の根拠となる |

## NIST（米国立標準技術研究所）が提唱する基本原則

1. データソースとコンピュータサービスをすべてリソースと見なす
2. ネットワークの場所を問わずすべての通信を保護する
3. 組織リソースへのアクセスは個別のセッション単位で許可する
4. リソースへのアクセスは動的なポリシーによって決定される
5. 組織が保有するすべてのデバイスに対して継続的に監視・計測する
6. リソースの認証と認可はアクセスが許可される前に動的かつ厳密に実施される
7. 資産・ネットワーク・通信の状態について可能な限り多くの情報を収集して利用する

## 構成要素

| 略称       | 正式名称                     | 役割                        |
| ---------- | ---------------------------- | --------------------------- |
| IAM        | IDアクセス管理               | 認証・認可の管理            |
| IAP        | ID認識型プロキシ             | アプリ単位のアクセス制御    |
| PKI        | 公開鍵認証基盤               | クラウドシステムの暗号化    |
| SWG        | セキュアWebゲートウェイ      | Webトラフィックの検査・制御 |
| TPM        | セキュリティチップ           | 端末レベルの暗号化・認証    |
| MDM・MAM   | モバイルデバイス管理         | 端末・アプリの管理          |
| DLP        | 情報漏洩防止                 | 重要情報の監視・漏洩防止    |
| SIEM・SOAR | セキュリティ情報イベント管理 | ログ収集・インシデント対応  |
